Four small tools

Focused utilities with explicit boundaries.

Diagnostic proxy Field evidence

MITMI

An in-path Modbus TCP diagnostic proxy. Clients connect to MITMI, MITMI connects upstream, and traffic is forwarded unchanged while logs, captures, analyzer summaries, and support bundles are produced.

  • Best for support sessions and traffic diagnosis.
  • Explicit client retargeting, not transparent interception.
  • Observed-value webhooks are disabled by default.
Download Repository
Address scanner Field validated

mitmi-scan

A read-only Modbus TCP address scanner for reverse engineering undocumented devices one address at a time, with console, CSV, and compact Markdown output.

  • Best for finding readable coils and registers.
  • Uses read-only Modbus functions only.
  • CSV keeps exhaustive evidence; Markdown stays compact.
Download Repository
TCP multiplexer Integration tested

mitmi-mux

A Modbus TCP-aware multiplexer that lets multiple downstream clients share upstream devices through configured routes while serializing requests and optionally caching repeated reads.

  • Best for devices that reject or suffer under concurrent clients.
  • Each route has one upstream connection and one request stream.
  • Writes are disabled by default and should be validated carefully.
Download Repository
CSV bridge Simulator tested

mitmi-csv

A localhost HTTP service that reads live Modbus TCP values and returns plain CSV for tools that already know how to fetch a URL.

  • Best for Excel Power Query, Power BI, browsers, and scripts.
  • No database, history, UI, JSON API, polling, or writes.
  • Real-device validation remains device and site specific.

Validation state

What has been proven, and where caution remains.

MITMI

Test suite and release smoke passed. Existing field evidence covers the Modbus TCP diagnostic proxy workflow. Webhook delivery remains best-effort and disabled by default.

mitmi-scan

Test suite, release smoke, and approved Teletek fire alarm panel validation passed for read-only address discovery.

mitmi-mux

Test suite and release smoke passed. The mux is covered by codec, route-runtime, cache, pacing, and NModbus integration tests; validate writes only against safe simulator or approved device addresses.

mitmi-csv

Test suite and published-service smoke passed. The Modbus read path is simulator-tested; validate against the actual device before a field-ready claim.
